# Short Game Design Document (SGDD)

## 1. Game Design Document (GDD)

- No início, o desenvolvimento de jogos era artesanal e feito por grupos pequenos.
- Com o crescimento do mercado e complexidade dos jogos, as equipes de desenvolvimento foram crescendo, exigindo centenas de profissionais especializados.
- Atualmente, o GDD é a principal ferramenta teórica para guiar o desenvolvimento de um jogo.
- Ele consiste em um documento de texto que descreve todos aspectos do jogo, como estética, roteiro, interface e mecânicas.
- É muitas vezes utilizado como uma bíblia para os envolvidos no desenvolvimento.
- Utilizado como ferramenta de consulta e também para informar possíveis investidores.
- Não existe um padrão a ser seguidos, geralmente os modelos de GDD são genéricos para que possam ser utilizados em diferentes jogos.
- São extensos, com muito textos, gráficos e imagens.

## 2. O problema do GDD

- O GDD é imprescindível em jogos Triple-A.
- Extremamente necessário para comunicação entre os diversos agente envolvidos no desenvolvimento.
- Em jogos pequenos, esses documentos extensos se mostram inadequados e inviáveis.
- Neles a equipe é reduzida e o prazo de entrega é muito curto.

## 3. SGGD

- Ferramenta textual que mapeia informações necessárias.
- Documento é escrito de forma que o leitor possa "jogar mentalmente" o jogo.

Possui 5 pilares:

**1.** Descrever de forma direta o enredo do jogo.

**2.** Descrever o jogo todo, do início ao fim, num texto corrido.

**3.** Marcar no texto com cores os elementos de arte, interface, música e mecânicas.

**4.** Criar uma lista contendo esses elementos para cada tópico.

**5.** Descrever o fluxo do jogo com desenhos
