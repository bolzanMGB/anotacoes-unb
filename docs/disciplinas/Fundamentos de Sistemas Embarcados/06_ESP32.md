# ESP32

## 1. Introdução


A ESP32 é uma MCU (Microcontroller Unit) dual-core de alta performance fabricada pela Espressif Systems. Ela é classificada como um microcontrolador porque integra, em uma única pastilha de silício, o processador, a memória (RAM/Flash) e os periféricos de entrada e saída. 

Diferentemente de microcomputadores como a Raspberry Pi (que executam sistemas operacionais pesados como Linux), a ESP32 possui recursos mais limitados e foi projetada especificamente para aplicações embarcadas e IoT, operando em Bare-metal ou com um RTOS.

## 2. Arquitetura de Hardware

A ESP32 possui:

### 2.1 Processador Dual-Core:

Possui dois núcleos independentes, ambos de 32 bits, baseados na arquitetura Xtensa LX6 e que operam com frequências de até 240 MHz:

**1. Core 0:** Dedicado ao processamento de infraestrutura de rede, gerindo as stacks do Wi-Fi, Bluetooth e tarefas críticas de sistema de baixo nível.

**2. Core 1:** Livre para rodar a aplicação principal desenvolvida pelo usuário (leituras de sensores, algoritmos e lógica de negócio).

### 2.2 Barramentos AMBA:

Faz a comunicação direta entre a CPU, as memórias (RAM e Flash), os controladores de periféricos e o módulo DMA (Direct Memory Access), permitindo que periféricos troquem dados direto com a memória sem sobrecarregar a CPU.

### 2.3 Memória:

Organizada da seguinte forma:

**1. Memória RAM:**

É a memória volátil, apagada ao desligar a ESP. Dividida em:

- DRAM (Data RAM): Armazena variáveis globais, variáveis estáticas, Heap, e Stacks de cada Task.
- IRAM (Instruction RAM): Armazena instruções que devem ser executadas em velocidade máxima, como as Rotinas de Interrupção (ISR).

**2. Memória Flash:**


É a memória não volátil, se mantém ao desligar a ESP. Dividida em:

- NSV (Non-Volatile Storage): Banco Chave/Valor para credenciais de rede, endereços de servidores, estados de calibração de sensores e etc.
- SPIFFS (SPI Flash File System): Banco plano para HTML, logs, arquivos JSon e configurações.
- OTA (Over-The-Air): Reservada para receber atualização de firmware via Wi-Fi.

### 2.4 Periféricos Internos

Blocos de hardware que podem ser utilizados pelo usuário via pinos GPIO. Alguns deles possuem GPIO Matriz, ou seja, possuem pinos default porém podem ser remapeados e associadas a quase qualquer pino. Outros são de fábrica associados a pinos fixos.

| Periférico   | Função                      |
| ------------ | --------------------------- |
| UART         | Comunicação serial          |
| I2C          | Comunicação serial síncrona |
| SPI          | Comunicação serial rápida   |
| PWM          | Controle de LEDs e motores  |
| ADC          | Conversão analógico→digital |
| DAC          | Conversão digital→analógico |
| CAN          | Rede industrial             |
| I2S          | Áudio digital               |
| Touch Sensor | Sensores capacitivos        |

## 3. Arquitetura de Software

**1. ESP-IDF (Espressif IoT Development Framework):**

A ESP-IDF é o framework de desenvolvimento oficial da fabricante. Ela atua como uma ponte intermediária entre o seu código e o hardware do chip. A ESP-IDF não é apenas uma biblioteca, mas sim um conjunto de ferramentas que engloba:

- Drivers Nativos: APIs escritas em C/C++ que abstraem os registradores do chip para controlar GPIO, ADC, DAC, I2C, etc.
- Pilhas de Rede (Stacks): Bibliotecas integradas para protocolos de conectividade, como a stack LwIP para Wi-Fi e a biblioteca esp-mqtt.
- FreeRTOS: Sistema Operacional embutido e configurado como o núcleo do framework.

**2. FreeRTOS:** 

Como dito, a ESP-IDF oferece um Sistema FreeRTOS que substitui o modelo clássico de loop infinito, como o do Arduino, por um ambiente Multi-tasking com:

- Tarefas (Tasks): O programa principal é fatiado em funções independentes chamadas Tasks, que são executadas de forma concorrente e assíncrona.
- Escalonador por Prioridade: O FreeRTOS gerencia os dois núcleos da ESP32, garantindo que as tarefas com maior prioridade configurada assumam o processamento imediatamente quando necessário, suspendendo as tarefas de menor prioridade.


**3. Menuconfig e Customização:** 

Ferramenta gráfica via terminal da ESP-IDF para ajustar variáveis de ambiente e parâmetros de hardware antes do código ser compilado:

- Comando `idf.py menuconfig`: Abre o terminal
- Arquivo `sdkconfig`: Guarda as escolhas feitas na interface gráfica.
- Header `sdkconfig.h`: Gerado pela leitura do sdkconfig. É referenciado internamente por todas as bibliotecas do sistema para moldar o comportamento do hardware.
- Menu `Kconfig.projbuild`: Menu personalizável do Menuconfig onde o desenvolvedor pode criar e configurar variáveis de compilação antes de gerar o firmware. É definido através do arquivo Kconfig.projbuild.
- Arquivo `sdkconfig.defaults`: Armazena configurações padrão de um projeto que devem ser compartilhadas entre diferentes desenvolvedores.

### 4.1 Tipos de Pinos

**1. Pinos GPIO Entrada/Saída (24):** Podem ser utilizados tanto como entrada ou saída. 

**2. Pinos GPIOEntrada (4):** Podem ser utilizados apenas como entrada. 

**3. Pinos GPIO Flash (6):** Podem ser utilizados tanto como entrada ou saída., porém são reservados memória Flash SPI onde o firmware está armazenado. Não são recomendados para o uso. 

**4. GND (1):** Referência de tensão 0V (Ground).

**5. 3V3 (1):** Referência de tensão 3.3V (VCC).

**6. vin (1):** Entrada de alimentação da placa (Voltage Input).

**7. GND (1):** Desliga/Reset (Enable).

## 5. Suportes Específicos de GPIO

Há algumas funcionalistas que são restritas à alguns pinos:

**1. GPIO ADC:** Podem receber tensão intermediária.

- ADC1: Funciona bem com WiFi.
- ADC2: Não Funciona bem com WiFi.

```bash
ADC1: GPIO032-GPI039
ADC2: GPIO0, GPI02, GPIO4, GPI012-GPIO015, GPI025-GPI027
```

**2. GPIO DAC:** Podem gerar intermediária.

```bash
DAC: GPIO025, GPI026
```

**3. GPIO Touch Sensor:** Podem detectar toque humano sem precisar de botão físico.

```bash
TOUCH: GPIO0, GPIO02, GPIO04, GPIO012-GPIO015, GPIO027, GPIO023, GPIO033
```

**4. GPIO RTC:** Podem manter noção de tempo (data e hora) mesmo:

- Desligado (modo sleep).
- Com CPU parada.

```bash
RTC: GPIO0, GPIO2, GPIO4, GPIO12–GPIO15, GPIO25–GPIO27, GPIO32–GPIO39
```

## 6. Dispositivos Internos

Em quais pinos posso usar certo dispositivo e quais possuem pinos "default"?

**1. UART:**

- Pode ser usado em qualquer pino de Entrada/Saída.
- Possui default, mas pode ser remapeada.
- Canais:
  - UART0: Reservada para PC.
  - UART1: Reservada para Flash.
- Pinos:
  - UART2: Livre para Dispositivos Externos.
  - TX: Saída de dados.
  - RX: Entrada de dados

```bash
UART0 Default
  TX -> GPIO1
  RX -> GPIO3

UART1 Default
  TX -> GPIO10
  RX -> GPIO9

UART2 Default
  TX -> GPIO17
  RX -> GPIO16
```

**2. I2C:**

- Pode ser usado em qualquer pino de Entrada/Saída.
- Possui default, mas pode ser remapeada.
- Pinos:
  - SDA: Entrada/Saída de dados.
  - SCL: Saída de clock.

```bash
I2C Default
  SDA -> GPI021
  SCL -> GPIO22
```

**3. SPI:**

- Pode ser usado em qualquer pino de Entrada/Saída.
- Possui default, mas pode ser remapeada.
- Canais:
  - VSPI: Não há diferença, sem restrição (Padrão).
  - HSPI: Não há diferença, sem restrição.
- Pinos:
  - MOSI: Saída de dados.
  - MISO: Entrada de dados.
  - CLK: Saída de clock.
  - CSO: Dispositivo

```bash
VSPI Default
  VSPI MOSI -> GPI023
  VSPI MISO -> GPI019
  VSPI CLK -> GPI018
  VSPI CSO -> GPI05

HSPI Default
  HSPI MOSI -> GPI012
  HSPI MISO -> GPI012
  HSPI CLK -> GPI014
  HSPI CSO -> GPI015
```

**4. PWM:**

- Pode ser usado em qualquer pino de Entrada/Saída.
- Não possui default, deve ser remapeada.
- Pinos:
  - PWM: Saída de dados.

**5. DAC:**

- Pode ser usado somente nos pinos DAC.
- Possui default, não pode ser remapeada (fixa).
- Canais:
  - DAC1: Não há diferença, sem restrição.
  - DAC2: Não há diferença, sem restrição.
- Pinos:
  - DAC: Saída de dados.

```bash
DAC1 
  DAC1 -> GPI025

DAC2 
  DAC2 -> GPIO26
```

**6. ADC:**

- Pode ser usado somente nos pinos ADC.
- Não possui default, pode ser remapeada.
- Pinos:
  - ADC: Entrada de dados.
