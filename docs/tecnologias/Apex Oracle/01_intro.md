## 3. Product Components

**1. App Builder:**

- Onde a aplicação realmente é criada.
- Como o VScode.
- É organizada em páginas.
- Cada página possui regions.
- Cada regions possui:
    - Items,
    - Buttons,
    - Reports,
    - Charts,
    - Forms.
    

**2. SQL Workshop:**

- Onde as tabelas são criadas.
- Como se fosse o pgAdmin do PostgreSQL.
- Pode:
    - Criar tabelas,
    - Criar views,
    - Importar dados,
    - Executar SQL.
- Possui as seguintes áreas:
    - 1. Object Browser:
        - Interface gráfica para tarefas comuns do banco de dados.
        - Adicionar e editar tabelas.
        - Adicionar e editar colunas.
        - Adicionar e editar linhas
    - 2. SQL Commands: 
        - Console para executar comandos SQL com acesso direto ao banco de dados.
        - create, edit, view, run e delete.
    - 3. SQL Scripts:
        - Armazena consultas que podem ser usadas.
        - Salva como arquivo. 
        - Tem como importar de arquivos.
    - 4. Utilities:
        - Ferramentas auxiliares na administração:
        - Data Workshop: 
            -Importar arquivos.
        - Quick SQL: 
            - Gera a estrutura do banco de dados (DDL).
            - Utilizando estrutura de pseuco-código.
        - Data Generator:
            - Gera dadis para preencher as tabelas (DML).
    - 5. REST Data Services:
        - Permite expor dados como APIs REST.
